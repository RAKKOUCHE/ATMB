T_LongAbsoluteCount Mei::Bnr::SystemUseHistory::SystemCycleCount
get

The number of system cycles.

Used to evaluate MCBF, MCBJ.

This counter increments (+1) every time :

  • Insertion cycle starts, Bill Intake motors start (Bill In).
  • A bill is extracted out of the system for a dispense (Bill Out).
  • A bill is transferred from the Loader to another module.
  • A bill is transferred from the Recycler to the Cashbox (Empty).
  • A reset command is executed.

When dispensing banknote, the counter is incremented every time:

  • A bill is moved from the Recycler or Loader to Bundler
  • A bill is moved from the Recycler or Loader to Outlet
  • A bundle is moved from the Bundler to Outlet This implies that:
  • When 1 bill is dispensed, systemCycleCount is incremented by 1
  • When N bills are dispensed (N > 1), systemCycleCount is incremented by N+1

Note. When transaction is canceled (CashInRollback), systemCycleCount is not incremented whatever the number of returned banknotes.
