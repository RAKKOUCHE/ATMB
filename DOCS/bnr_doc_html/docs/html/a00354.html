void Mei::Bnr::Bnr::ResetUsbDevice ( )

This function is used to reset the Device port.

All the pending transactions on the bus should be aborted. The request causes all of the status and configuration values associated with endpoints in the affected interfaces, to be set to their default values.

Returns
nothing if successful.
void Mei::Bnr::Bnr::ResetUsbPipe ( int  endPoint)

Resets the endpoint and its corresponding pipe.

This function resets a stalled pipe. It cancels all pending I/O on the pipe and sends CLEAR_FEATURE with USB_FEATURE_ENDPOINT_STALL selector for the specified endpoint. When receiving stallPid error code on bulk or interrupt transfers, the Client application should try to reset the pipe. If this does not help, the Client should try to reset the Device.

The USB communication for the BNR uses 4 endpoints other than the control endpoint(0) :

  • Endpoint 1 defines has input to receive the method response from the BNR.
  • Endpoint 2 defines has output to send the method call to the BNR.
  • Endpoint 3 defines has interrupt to receive the method call from the BNR.
  • Endpoint 4 defines has output to send the method response to the BNR.
Parameters
[in]endPointNumber of the endpoint to reset.
Returns
nothing if successful.
void Mei::Bnr::Bnr::SetDeviceChangeListener ( HWND  listener)

Registers a specified window handle has listener of the USB device change notifications.

The notification becomes as the WM_DEVICECHANGE window message, where the lParam parameter points to the buffer with DEV_BROADCAST_DEVICEINTERFACE structure, from which the Client application can extract required fields about notification.

Parameters
[in]listenerspecifies window handle for which notification enables.
Returns
nothing if successful.
