Detailed Description

Those functions are delegates that need to be implemented in the application that is using the API.

They are used to fire event from the BNR.

Example on how to use Bnr Events
// Bnr object creation
Bnr bnr = new Bnr();
// Listen for events
Bnr.OperationCompletedEvent += new OperationCompleted(Bnr_OperationCompletedEvent);
Bnr.StatusOccuredEvent += new StatusOccured(Bnr_StatusOccuredEvent);
Bnr.IntermediateOccuredEvent += new IntermediateOccured(Bnr_IntermediateOccuredEvent);
Bnr.CashOccuredEvent += new CashOccured(Bnr_CashOccuredEvent);
// Connect on Bnr
bnr.Open();
// Reset
bnr.Reset();
// CashIn functions
bnr.CashInStart();
bnr.CashIn(100, "AAA");
bnr.CashInRollback();
bnr.CashInEnd();
// Closing Bnr
bnr.Close();

OperationId  operationId,
IntermediateEvents  result,
System::Object^  Data 
)

Raised on Intermediate event reception.

Parameters
identificationIdThe id which was given by the operation method to the application, -1 if not used.
operationIdThe operationID which states what kind of asynchronous operation is running.
resultSpecifies what the reason for this event was (e.g. InputRefused).
DataContains optionally added data for the application or null.

Depending on result parameter value, Data parameter has different meanings, and must be casted as follows :

  • if result is InputRefused, if Data is not null, it is a uint containing internal reason code.
  • if result is IntermediateEvents.SubCashin, if Data is not null, it is a XfsCashOrder object.

public delegate void Mei::Bnr::StatusOccured ( StatusChanged  status,
BnrXfsErrorCode  result,
BnrXfsErrorCode  extendedResult,
System::Object^  Data 
)

Raised on Status event reception.

Parameters
statusThe status the device has changed to. The appropriate StatusChanged enum value.
resultAlways set to NotUsed .
extendedResultAlways set to NotUsed .
DataContains information about current device status, depending on status parameter value.

Depending on status parameter value, Data parameter has different meanings, and must be casted as follows :

  • if status is DeviceStatusChanged and Data is not null, it is a pointer to a uint value set to a DeviceStatus value.
  • if status is CashAvailable or CashTaken, if Data is not null, it is a pointer to a XfsCashOrder object.
  • if status is CashUnitChanged, CashUnitConfigurationChanged or CashUnitThreshold, Data is a pointer to a XfsCashUnit object containing the LCU and PCU that have changed.
  • if status is MaintenanceStatusChanged and Data is not NULL, it is a pointer to a list containing modules that maintenance is recommended.
See also
XfsCashOrder, StatusChanged, BnrXfsErrorCode.
