enum Mei::Bnr::DeviceType
strong

DeviceType values as defined by J/Xfs.

See also
Capabilities::CdType.
Enumerator
Dispenser 

Device dispensing cash.

Consist of components that allow the dispensing of cash, either bills or coins. This interface provides common functionality that is although used by the following device types.

Recycler 

Device accepting and dispensing cash.

A Recycler is primarily a Dispenser plus additional components that allow acceptance of cash as input to the device. This specification for Recyclers is intended for branch-teller environments and not for use in self-service environments.

ATM 

Device that inherit from Recycler with additional functions to support ATM-specific hardware.

The BNR is part of this family.

enum Mei::Bnr::SelfTestMode
strong

Defines how the BNR perform the self tests.

Integration recommendations:
If the slave mode is used, the Host should send a Bnr::Bnr::SelfTest() command at every time that internal tests and movements of the BNR can be allowed.
It is recommended to send Bnr::Bnr::SelfTest(), after a Bnr::Bnr::Present() command to allow the BNR to refloat the recyclers.
See also
Capabilities::SelfTestMode, Bnr::Bnr::SelfTest().
Enumerator
Auto 

The BNR does self tests automatically (default and recommended mode).

After Bnr::Bnr::Reset(), Bnr::Bnr::Present() command only, the BNR refloats the Recyclers from the Loader, when necessary.

Slave 

The BNR doesn't do any test automatically.

It waits for a Bnr::Bnr::SelfTest() command to do all self test actions.

STM_AUTO 
Deprecated:
Use Auto.
STM_SLAVE 
Deprecated:
Use Slave.

array< Mei:: Bnr:: StatusChanged>^ Mei::Bnr::Bnr::EnabledStatusEvents
getset

List of status events enabled in the BNR.

Those settings are not persistent over power cycles, and must be set on power up. If not set, by default on power up, all events defined prior to FW v1.7.0 are enabled and all new events introduced in this version and later, are disabled to preserve the legacy integrations. In other words:

  1. default enabled status events are:
    • CashUnitChanged
    • CashUnitConfigurationChanged
    • CashUnitThreshold
    • DeviceStatusChanged
    • CashTaken
    • CashAvailable
  2. default disabled status events are:
    • MaintenanceStatusChanged
Exception :
  • ParameterInvalid - The list is not valid (Set only), ie: one of the status event is unknown.
  • NotSupported - The property is not suppoted by the current firmware.
Firmware Compatibility
This propertiy requires a BNR FW v1.7.0 or newer. With older FW versions, a NotSupported exception will be raised.
See also
Bnr::StatusChanged for exhaustive event list.
BnrXfsErrorCode, BnrApiErrorCode, BnrUsbErrorCode for more exception details.
