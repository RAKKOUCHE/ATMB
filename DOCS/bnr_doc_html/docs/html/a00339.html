int Mei::Bnr::Bnr::AddDenomination ( String^  pathFileName)

Add a denomination to the BNR.

This function will download denomination data from a DLD file to the BNR. If the billTypeDescriptor is already present on the BNR,this will raise an error. If the bill already exists, then this command updates the bill. The LCU are not modified. In the case of a new bill, an LCU Deposit is automatically created on the cashBox. This method puts the BNR in an IDLE state. A reset is then mandatory to put the BNR in a working state.

Note
This method puts the BNR in an IDLE state. A reset is then mandatory to put the BNR in a working state.
Parameters
[in]pathFileNamePath and filename to download
Returns
The identification Id if successful.
Exception:
  • Mei::Bnr::BnrXfsErrorCode::ParameterInvalid - No more denomination can be added.
  • Illegal - The BNR is in a state where it cannot execute the operation, e.g. another asynchronous command is already running, a dispense is active (after a Bnr::Dispense() and before a Bnr::Present() call), etc...
StatusEvent:
DeviceStatusChanged is fired as soon as this command is accepted by the system.
OperationCompleted event:
The field data is always NULL and result field can take the following values :
  • Successful - The Command has been executed successfully
  • TooManyBills - The intermediate stacker is not empty (e.g. if, during a refloat cycle, a jam occurred or the cash modules lock has been opened) and thus no denomination can be added at the moment.
See also
BnrXfsErrorCode, BnrApiErrorCode, BnrUsbErrorCode for more exception details.
int Mei::Bnr::Bnr::DeleteDenomination ( String^  currencyCode,
int  value,
String^  variant 
)

Deletes a denomination in the BNR.

The method needs the currencyCode, the value and the variant.

Note
The intermediate stacker must be empty in order to delete denominations. After this command, the BNR is in OffLine state. A Bnr::Reset() command is needed to put it in OnLine state.
Parameters
[in]currencyCodeCurrency code of the denomination to delete.
[in]valueValue of the denomination to delete. Expressed in MDU.
[in]variantVariant of the denomination to delete.
Returns
The identification Id if successful.
Exception:
  • ParameterInvalid - No denomination corresponding to the parameters was found in the BNR.
  • CashInActive - A transaction started with Bnr::CashInStart() is already in progress and must be ended with Bnr::CashInEnd() before calling Bnr::DeleteDenomination().
  • Illegal- The BNR is in a state where it cannot execute the operation, e.g. another asynchronous command is already running, a dispense is active (after a Bnr::Dispense() and before a Bnr::Present() call), etc...
Other exceptions might be used in the future without notice.
OperationCompleted event:
The field data is always NULL. The filed result can take the following values:
  • Successful - The command has been executed successfully.
  • Illegal - The extendedResult param contains more details :
    • TooManyBills - The intermediate stacker is not empty and thus the denomination can not be deleted.
    • ParameterInvalid - No denomination corresponding to the parameters was found in the BNR.
See also
BnrXfsErrorCode, BnrApiErrorCode, BnrUsbErrorCode.
int Mei::Bnr::Bnr::UpdateDenomination ( array< Mei::Bnr::XfsDenominationInfo^ >^  infoDotNet)

This function is used to update the settings for a list of denominations.

For each element of the infoDotNet, the application can update its validation settings.

Those settings are persistent over power cycles; please refer to XfsDenominationInfo for more details about settable properties, and their default values.

Parameters
infoDotNetarray of denomination
Returns
The identificationId if successful or the following error code :
  • Illegal - A dispense command is already active on the BNR.
  • NotSupported - operation not supported by the BNR firmware version.
  • ParameterInvalid - Invalid array size. The array size is bigger than expected.
  • CashInActive - A cashIn command has been issued and is already active.
  • Failure - a command is already running on the BNR or an internal error occured.
