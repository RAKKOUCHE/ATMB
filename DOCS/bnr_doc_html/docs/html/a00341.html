int Mei::Bnr::Bnr::Park ( )

Asynchronous command which prepares all modules in the BNR to be removed.

Returns
If function call is successful, return is strictly positive and represents the identificationId.
Exception:
  • CashInActive - A transaction started with Bnr::CashInStart() is already in progress. The transaction must be ended with Bnr::CashInEnd().
  • Illegal - Another asynchronous command is already running, or a dispense is active (after a Bnr::Dispense() and before a Bnr::Present() call).
  • Failure - No "parkable" module was found in the BNR.
OperationCompleted event:
result field can take the following values :
  • Successful - The command has been executed successfully.
  • CashDeviceError - A failure occured during some module movement.
  • Canceled - The command has been interrupted because:
    • The Cash Modules Lock or a cover has been opened.
    • Bnr::Cancel() has been called while bills were being presented to the user.
See also
BnrXfsErrorCode, BnrApiErrorCode, BnrUsbErrorCode for more exception details.
void Mei::Bnr::Bnr::PreventativeMaintenanceCompleted ( PreventativeMaintenanceLevel  pmLevel)

Indicates to the BNR that the Preventative Maintenance (PM) operation of the specified level was completed.

This will update the date/time and systemCycleCount information of the specified level with current date/time and current systemCycleCount.

Parameters
[in]pmLevelLevel of Preventative Maintenance that has been completed.
Exception :
  • ParameterInvalid - The Preventative Maintenance level is out of range.
  • Busy - A transaction is in progress.
  • CashInActive - A transaction started with Bnr::CashInStart() is already in progress. The transaction must be ended with Bnr::CashInEnd()
  • Illegal - The state of the BNR does not allow this command (e.g. asynchronous command running, cash module lock open, module locked, etc...).
  • Failure - The command un-succeeded due to an internal failure.
Other error codes might be used in the future without notice.
