void Mei::Bnr::Bnr::ConfigureCashUnit ( CashUnit::CashUnit^  cashUnit)

This function is used to manually add or delete BNR cashUnit, to change the LCU's cashType and other cashUnit configuration options.

After calling ConfigureCashUnit the cashUnit instance is updated with the cashUnits available in the BNR.

Those settings are persistent over power cycles.

Parameters
[in,out]cashUnitNew cash units to be applied in the BNR. On return, this instance contains the cashUnit available in the BNR after the call.
Exception:
  • Busy - The BNR is moving bill or about to move and cannot update CashUnit information.
  • Illegal- The BNR is in a state where it cannot execute the operation, e.g. a dispense transaction is active in the BNR, the Cash Modules Lock is opened, etc...
  • ParameterInvalid - There are inconsistencies between LCU list and PCU list.
  • CashInActive - A transaction started with Bnr::CashInStart() is already in progress and must be ended with Bnr::CashInEnd() before calling Bnr::ConfigureCashUnit().
  • TooManyBills - No transaction is active, but the BNR has bills in escrow, or in its transport system.
  • InvalidCashUnit - An invalid Cash Unit information was sent to the BNR; typically, this error can happen when :
    • an invalid PCU name is passed,
    • a PCU/LCU is listed more than once in the PCU/LCU list.
  • CashUnitError - An unauthorized Cash Unit modification was requested; typically , this error can happen when :
    • a PCU bill count doesn't match with the sum of all LCU bill counts attached to this LCU,
    • a Recycler PCU/LCU count doesn't match with the counts inside the BNR,
    • a Recycler or Bundler PCU threshold mode is set to SensorMode instead of CountMode,
    • the Bundler PCU threshold levels don't match with the levels inside the BNR,
    • attempting to lock the Bundler or Cashbox PCU.
    • If more than one cash type (note series or denomination) is configured to the same cash unit AND the differential length among any two > 5 mm (LO and RE)
    • If more than one cash type (note series or denomination) is configured to the same cash unit AND the differential width among any two is > 1 mm (LO only)
See also
Bnr::UpdateCashUnit(), CashUnit, BnrXfsErrorCode, BnrApiErrorCode, BnrUsbErrorCode.

Example on how to delete Recycler 3 PhysicalCashUnit with ConfigureCashUnit

try{
foreach (PhysicalCashUnit pc in ret.PhysicalCashUnits){
if (pc.Name == "RE3"){
found = true;
break;
}//if
}//foreach
if (found == false){
System.Console.WriteLine("RE3 does not exist.\n");
}//if
foreach (PhysicalCashUnit PCU in ret.PhysicalCashUnits){
if (PCU.Name == "RE3"){
ret.PhysicalCashUnits.Remove(PCU);
break;
}//if
}//foreach
Bnr.ConfigureCashUnit(ret);
}catch (BnrUsbException usbEx){
System.Console.WriteLine(usbEx.Message.ToString() + " in " + usbEx.TargetSite.ToString());
}catch (BnrXfsException xfsEx){
System.Console.WriteLine(xfsEx.Message.ToString() + " in " + xfsEx.TargetSite.ToString());
}catch (BnrApiException apiEx){
System.Console.WriteLine(apiEx.Message.ToString() + " in " + apiEx.TargetSite.ToString());
}//try-catch

Another Example on how to create RE3

//retrieving cashunits
ret = bnr.CashUnit;
//PCU config
PhysicalCashUnit pcu = new PhysicalCashUnit();
pcu.Name = "RE3";
ret.PhysicalCashUnits.Add(pcu);
//Lcu1 config
LogicalCashUnit lcu1 = new LogicalCashUnit();
CashType ct = new CashType();
ct.CurrencyCode = "AAA";
ct.Value = 1000;
ct.Variant = "AA";
lcu1.CashType = ct;
lcu1.CuKind = CashUnitKind.Recycle;
lcu1.CuType = CashUnitType.BillCassette;
lcu1.PhysicalCashUnit = pcu;
lcu1.PhysicalName = pcu.Name;
ret.LogicalCashUnits.Add(lcu1);
//Lcu2 config
LogicalCashUnit lcu2 = new LogicalCashUnit();
CashType ct2 = new CashType();
ct2.CurrencyCode = "AAA";
ct2.Value = 1000;
ct2.Variant = "AA";
lcu2.CashType = ct;
lcu2.CuKind = CashUnitKind.NA;
lcu2.CuType = CashUnitType.Escrow;
lcu2.PhysicalCashUnit = pcu;
lcu2.PhysicalName = pcu.Name;
ret.LogicalCashUnits.Add(lcu2);
Bnr.ConfigureCashUnit(ret);
void Mei::Bnr::Bnr::ResetCashboxCuContent ( bool  resetDepositCounters)

Resets counters in cashbox.

It resets Cashbox PCU+LCUs bill count (+initialCount). If resetDepositCounters parameter sets to true, then it`ll reset extended counters.

Parameters
[in]resetDepositCountersIf False, only Cashbox PCU + LCUs bill count (+ initialCount) are resetted, otherwise depositCounters are also resetted.
Exception:
  • NoMemory - Not enough memory to query cashUnit
  • Busy - The BNR is moving bill or about to move and cannot update CashUnit information.
  • Illegal- The BNR is in a state where it cannot execute the operation, e.g. a dispense transaction is active in the BNR, the Cash Modules Lock is opened, etc...
  • CashInActive - A transaction started with Bnr::CashInStart() is already in progress and must be ended with Bnr::CashInEnd() before calling Bnr::ResetCashboxCuContent().
See also
Bnr::UpdateCashUnit(), CashUnit, BnrXfsErrorCode, BnrApiErrorCode, BnrUsbErrorCode.
