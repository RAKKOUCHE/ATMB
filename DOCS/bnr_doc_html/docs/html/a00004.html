BNR concepts

These are the standard application layers defined by MEI.

The host application will communicate through the BNR API for Win32 with the BNR.

The BNR API for .NET needs 6 files to work properly:

  • BnrCtlDotNet.dll : Provides .NET class interfaces to communicate with the BNR.
  • BnrXmlRpc.dll: Library related to BNR XML-RPC.
  • XmlParse.dll: Library related to XML.
  • XmlTok.dll: Library related to XML.
  • BnrCtlW32.dll: Containing the functions to manage the communication between the host application and the BNR.
  • BnrCtl.ini: Configuration file for the logs files and other options.
Events
BNR API for .Net delivers information to the application by events. Events are used by the asynchronous methods and can be sent at any time. The following duties are assigned to the events:
  1. Notify the application of intermediate results during the running operation.
  2. Notify the application of asynchronous operation completion. This may be successful completion as well as abortion of the operation due to an error.
  3. Notify the application of status changes and special conditions.
To satisfy the above duties the following categories of events exist:
  1. The IntermediateEvent (I) is sent whenever meaningful intermediate result is available for the running asynchronous operation. As it can occur only during a cash in, see Mei::Bnr::Bnr::CashIn() for the different possible IntermediateEvents (see also Mei::Bnr::Bnr::IntermediateOccured()).
  2. The OperationCompleteEvent (OC) is sent whenever an asynchronous operation is completed. The parameters passed to the event being dependent on the current asynchronous operation, one should refer to the operation documentation to get details about the possible OperationCompleteEvent values (see also Mei::Bnr::Bnr::OperationCompleted()).
  3. The StatusEvent (S) is sent whenever a BNR's element status changes. Some StatusEvents can occur only if an asynchronous operation is running (e.g. Mei::Bnr::StatusChanged::CashAvailable), and others can occur at any time (e.g. Mei::Bnr::StatusChanged::DeviceStatusChanged) (see also Mei::Bnr::Bnr::StatusOccured()).
