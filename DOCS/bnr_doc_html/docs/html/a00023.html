Detailed Description

Capabilities.

By default, capabilities are read-only properties, but the following ones are writable :

  • AutoPresent,
  • SelfTestMode,
  • AntiFishingLevel,
  • AllowUsbFrontSwitch,
  • ReportingMode.
  • ReportUsbConsumption.
  • AutoRetract.
  • RejectViaOutlet.
See also
Bnr::Bnr::Capabilities.

Property Documentation

bool Mei::Bnr::Capabilities::AllowUsbFrontSwitch
getset

Allows to use USB Front interface to communicate with the BNR.

Default value is TRUE.

Mei:: Bnr:: AntiFishingLevel Mei::Bnr::Capabilities::AntiFishingLevel
getset

Sensitivity level of string detection at Inlet.

Default value is Normal.

bool Mei::Bnr::Capabilities::AutoPresent
getset

This specifies whether cash will be automatically presented to the user on execution of a dispense (autoPresent set to TRUE), or whether the cash will only be transported to the Bundler.

In the latter case, a Bnr::Present() command will need to be issued following the Bnr::Dispense() command. Default value is FALSE.

bool Mei::Bnr::Capabilities::AutoRetract
getset

This specifies whether bill will be automatically retracted to positioner/BTspine when jam occured during bill presenting at inlet/outlet.

Default value is FALSE.
