T_LongAbsoluteCount Mei::Bnr::Module::MaintenanceInfo::CycleSinceLastMaintenance
get

Number of module cycles since the last maintenance operation.

Resetted to 0 on Module::MaintenanceDone() call.

UINT16 Mei::Bnr::Module::MaintenanceInfo::MaintenanceBundlerSlippage
get

For bundler module type, this is the slippage value in 1/10 %.

E.g a returned value of 25 means 2.5% of slippage. For other module types, it remains to 0.

Firmware Compatibility
This propertie requires a BNR FW v1.9.0 or newer. With older FW versions, the value will remain to 0.
T_LongAbsoluteCount Mei::Bnr::Module::MaintenanceInfo::MaintenanceInterval
getset

Number of module cycles between two maintenance operations.

Default value is 50000 for all modules, except for Loader (20000).

Those settings are persistent over power cycles.

Exception (Set only):
  • ParameterInvalid - The module is not valid.
  • CashInActive - A Bnr::Bnr::CashInStart() command has been issued and is already active.
  • Illegal - The state of the BNR does not allow this command (e.g. asynchronous command running, cash module lock open, etc...).
  • NoHardware - The module is missing.
  • Failure - The command un-succeeded due to an internal failure.
  • NotSupported - The property is not suppoted by the current firmware.
StatusEvent event (Set only):
The field status can take the following values :
  • MaintenanceStatusChanged - if the new interval modifies the MaintenanceInfo::MaintenanceStatus.
